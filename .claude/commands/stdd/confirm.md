# /stdd:confirm - 需求确认

用户审阅并确认需求，作为进入规格编写阶段的人机确认门。

## 使用方式

```
/stdd:confirm                  # 确认当前活跃变更
/stdd:confirm --change=<name>  # 确认指定变更
```

## 执行流程

1. **读取提案** - 加载 `proposal.md`（含原始需求 + 澄清记录）
2. **生成确认报告** - 汇总原始需求、澄清答案、BDD 大纲预览
3. **等待用户确认** - 提示输入 `yes` 或 `no`
4. **分支处理**：
   - **yes** → 追加 `<!-- Confirmed -->` 标记，自动进入 `/stdd:spec`
   - **no** → 追加 `<!-- Rejected -->` 标记，返回 `/stdd:propose` 重新描述

## 确认门控

```
┌─────────────────────────────────────────┐
│           Human Confirm Gate             │
├─────────────────────────────────────────┤
│                                          │
│  📋 需求确认报告                          │
│  ├── 原始需求: ...                       │
│  ├── 澄清记录: ...                       │
│  └── BDD 预览: ...                       │
│                                          │
│  确认？(yes/no)                          │
│                                          │
│  yes → ✅ 进入 /stdd:spec               │
│  no  → ❌ 返回 /stdd:propose            │
└─────────────────────────────────────────┘
```
