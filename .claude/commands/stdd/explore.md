# /stdd:explore - 自由探索模式

进入只读探索模式，在动手前理清思路，可衔接 `/stdd:new`。

## 使用方式

```
/stdd:explore [探索目标]
```

## 特点

- **只读模式** - 不修改任何代码
- **自由思考** - 允许探索多个方向
- **产出分析** - 生成探索报告
- **可衔接** - 探索结果可直接用于创建提案

## 执行流程

1. **分析现有代码** - 理解当前架构
2. **识别约束** - 技术栈、框架、规范
3. **探索方案** - 多种实现可能性
4. **生成报告** - 包含发现和建议

## 输出文件

```
stdd/changes/explorations/
└── exploration-YYYYMMDD-HHMMSS.md
```

## 示例

```
/stdd:explore 理解现有的认证系统架构

/stdd:explore 分析数据库访问模式

/stdd:explore 评估引入 GraphQL 的可行性
```

## 适用场景

- 需求表达不出来时
- 对现有系统不了解时
- 需要技术选型时
- 想要评估多个方案时
