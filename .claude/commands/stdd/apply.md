# /stdd:apply - 实现任务

执行 `tasks.md` 中定义的任务，遵循 TDD 循环。

## 使用方式

```
/stdd:apply                    # 执行所有待办任务
/stdd:apply --task=TASK-001    # 执行特定任务
/stdd:apply --next             # 执行下一个任务
/stdd:apply --fix              # 修复失败的测试
```

## Ralph Loop (TDD 循环)

```
┌──────────────────────────────────────────────────────┐
│                    Ralph Loop                         │
│                                                     │
│  🔴 红灯      →    🔍 静态检查    →    🟢 绿灯       │
│  生成失败测试       语法/类型检查       最简实现       │
│                                                     │
│       →    🧪 伪变异审查    →    🔵 重构    →    ✅   │
│           检测骗绿灯断言       优化代码                  │
│                                                     │
│  ⚠️ 连续失败 3 次 → 自动熔断                            │
└──────────────────────────────────────────────────────┘
```

## 执行流程

1. **读取任务列表** - 从 `tasks.md` 获取待办任务
2. **红灯阶段** - 先写失败测试
3. **静态检查** - 语法、类型、规范检查
4. **绿灯阶段** - 最小实现通过测试
5. **变异测试** - 检测骗绿灯断言
6. **重构阶段** - 优化代码
7. **更新状态** - 标记任务完成

## 任务状态

| 状态 | 标识 | 说明 |
|------|------|------|
| 待办 | `[ ]` | 未开始 |
| 进行中 | `[🔄]` | 正在执行 |
| 完成 | `[x]` | 已完成 |
| 阻塞 | `[⚠️]` | 需要帮助 |
| 跳过 | `[~]` | 已跳过 |

## 示例

```bash
# 执行所有任务
/stdd:apply

# 只执行 TASK-001
/stdd:apply --task=TASK-001

# 执行下一个任务
/stdd:apply --next

# 修复模式 - 自动修复失败测试
/stdd:apply --fix
```

## 5级防跑偏防御

1. **人机确认门** - 关键决策需确认
2. **微任务隔离** - 5~6 个原子任务
3. **连续失败回滚** - 3 次失败熔断
4. **静态质检门** - 语法/类型检查
5. **伪变异审查** - 检测骗绿灯
