# /stdd:plan - 任务拆解

评估架构变更并生成细粒度的微任务清单白板。

## 使用方式

```
/stdd:plan                     # 为当前活跃变更拆解任务
/stdd:plan --change=<name>     # 为指定变更拆解任务
```

## 执行流程

1. **契约对齐** - 在 `stdd/memory/contracts.md` 中确定跨模块数据结构与接口类型
2. **架构思考** - 在 `design.md` 中画出组件依赖流与架构拓扑
3. **拆解微任务** - 在 `tasks.md` 中用 Markdown Checkbox 制作极细步骤
4. **确认门控** - 通知用户签批任务清单
5. **结束引导** - 用户认同后，提示输入 `/stdd:apply` 开始 TDD 执行

## 任务拆解原则

- 每个变更最多 5-6 个原子任务（防跑偏）
- 由外向内推进：E2E → 底层核心 → 业务胶水层 → UI/CLI
- 每个任务预估 30 分钟内完成

## 任务格式

```markdown
## Tasks

- [ ] TASK-001: <任务描述>
  - 输入: <依赖>
  - 输出: <产物>
  - 测试: <测试文件>

- [ ] TASK-002: <任务描述>
  ...
```
